## 4. Fluxo Completo: Adicionando um Use Case de Leitura (Query)

Neste projeto NÃO usamos CQRS formal. Queries são use cases simples sem Command.

Subseção 4.1 — Padrão de query sem CQRS:

Mostrar o padrão recomendado com snippet mínimo ilustrativo (não um arquivo existente, pois não
existe ainda — criar o exemplo canônico para o guia):

```java
// aplicacao/BuscarMatriculaUseCase.java
@Service
public class BuscarMatriculaUseCase {

    private final MatriculaRepositorio repositorio;

    public BuscarMatriculaUseCase(MatriculaRepositorio repositorio) {
        this.repositorio = repositorio;
    }

    public MatriculaDto executar(UUID matriculaId) {
        Matricula matricula = repositorio.buscarPorId(matriculaId)
            .orElseThrow(() -> new MatriculaNaoEncontradaException(matriculaId));
        return MatriculaDto.de(matricula);
    }
}
```

Subseção 4.2 — Adicionar método ao repositório (interface de domínio primeiro):

Mostrar que ao adicionar uma query, o fluxo é:
1. Adicionar assinatura na interface `MatriculaRepositorio` (dominio/)
2. Implementar em `MatriculaRepositorioMyBatis` chamando o Mapper
3. Adicionar método no `MatriculaMapper` (@Mapper)
4. Adicionar `<select>` no `MatriculaMapper.xml`

Subseção 4.3 — Quando usar query direta no Mapper vs. passar pelo Repositório:

Regra simples: SEMPRE passar pelo Repositório. O Mapper é detalhe de infraestrutura.
UseCases nunca injetam `@Mapper` diretamente.

## 5. MyBatis: Pitfalls Críticos

Esta seção usa exemplos reais do `MatriculaMapper.xml`.

**Pitfall 1 — UUID PostgreSQL exige `jdbcType=OTHER`:**
```xml
<!-- ERRADO — causa "operator does not exist: uuid = character varying" -->
WHERE id = #{id}

<!-- CERTO -->
WHERE id = #{id, jdbcType=OTHER}
```
Regra: todo parâmetro UUID no XML precisa de `jdbcType=OTHER`.

**Pitfall 2 — LEFT JOIN sem `notNullColumn` cria objetos fantasma:**
```xml
<!-- ERRADO — matrícula sem itens gera ItemMatriculaRow(disciplina=null) na lista -->
<collection property="itens" ofType="ItemMatriculaRow">

<!-- CERTO — lista fica vazia quando não há itens -->
<collection property="itens" ofType="ItemMatriculaRow" notNullColumn="item_disciplina">
```
Regra: SEMPRE usar `notNullColumn` em `<collection>` com LEFT JOIN.

**Pitfall 3 — Construtor de reconstituição precisa inicializar `eventos`:**
Ao adicionar novo Aggregate com lista de eventos, o construtor de reconstituição (usado pelo
RowMapper) DEVE incluir `this.eventos = new ArrayList<>()`. Sem isso, `adicionarDisciplina()`
lança NullPointerException. Ver Matricula.java — construtor público de reconstituição.

**Pitfall 4 — Domain Service sem `@Service`:**
Domain Services no pacote `dominio/` não têm `@Service`. São configurados como `@Bean` em
`infraestrutura/config/`. Ver `DomainServicesConfig.java`.

## 6. Value Objects: Padrão com Java 21 Record

Mostrar o padrão mínimo de VO com validação no construtor compacto:

```java
// dominio/vo/NovoVO.java
public record NovoVO(String valor) {
    public NovoVO {
        if (valor == null || valor.isBlank()) {
            throw new IllegalArgumentException("NovoVO não pode ser vazio");
        }
        valor = valor.trim(); // normalização no construtor compacto
    }
}
```

Linha: "Igualdade por valor é automática com `record` — não implementar `equals`/`hashCode` manualmente."

Referência para o padrão com validação mais complexa: `dominio/vo/Cpf.java` (dígito verificador).

## 7. Schema PostgreSQL: Convenções

Regras resumidas (detalhes em V1__schema.sql e ADR-003):
- Uma tabela por Aggregate Root + tabelas de entidades internas (com FK + ON DELETE CASCADE)
- Referências a outros Aggregates: apenas UUID, SEM FOREIGN KEY (ADR-003)
- Status como `VARCHAR(20)` + `CHECK` constraint — não usar enum PostgreSQL
- UUID como tipo nativo — não `VARCHAR(36)`
- Migrações em `src/main/resources/db/migration/` com prefixo `V{N}__descricao.sql`
