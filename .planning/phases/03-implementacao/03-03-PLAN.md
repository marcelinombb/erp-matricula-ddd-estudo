  Anotações: @Service, @Transactional (org.springframework.stereotype.Service, org.springframework.transaction.annotation.Transactional)

    private final VerificadorElegibilidadeMatricula verificador
    private final ApplicationEventPublisher publicador (org.springframework.context.ApplicationEventPublisher)
  Construtor único com todos os três campos.

  Método público MatriculaId executar(MatricularAlunoCommand command):
    Sequência OBRIGATÓRIA per D-10 (comentar cada passo no código):
    // 1. Validar elegibilidade (Domain Service — lança exceção se não elegível)
    verificador.verificar(command.aluno(), command.turma(), command.periodo());

    // 2. Criar Aggregate (domínio decide — estado inicial ATIVA, evento AlunoMatriculado coletado)
    Matricula matricula = Matricula.criar(command.aluno().getId(), command.turma().getId(), command.periodo());

    // 3. Persistir ANTES de publicar eventos (D-10: ordem obrigatória)
    repositorio.salvar(matricula);

    // 4. Publicar eventos APÓS persistência
    // @Transactional + @TransactionalEventListener garante que listeners só executam após commit
    // Sem esta ordem, um rollback posterior processaria eventos de dados não persistidos
    matricula.coletarEventos().forEach(publicador::publishEvent);

    return matricula.getId();

  Javadoc na classe: "Application Service — orquestra sem decidir. Valida elegibilidade (Domain Service), cria Aggregate (domínio), persiste (Repositório), publica eventos (D-10). Não contém lógica de negócio: toda regra fica no Aggregate ou no Domain Service."

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Command → UseCase | Commands carregam objetos de domínio (Aluno, Turma) vindos de fora do UseCase |
| UseCase → ApplicationEventPublisher | Eventos publicados para listeners que podem executar ações de efeito colateral |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | MatricularAlunoCommand (carrega objetos Aluno e Turma) | accept | Na Fase 3 o caller é DemoRunner (código confiável). Na Fase 4, o Controller validará o Command antes de criar os objetos de domínio. Objetos de domínio têm validação no construtor. |
| T-03-05 | Denial of Service | ApplicationEventPublisher.publishEvent() em loop | accept | O loop é sobre a lista de eventos coletados do próprio Aggregate. O Aggregate só adiciona eventos nos métodos de negócio — número de eventos é proporcional às operações, não a input externo. |
| T-03-SC | Tampering | org.springframework dependências nesta camada | accept | Spring Boot BOM gerencia versões; spring-boot-starter-parent 3.5.3 está verificado. @Transactional é Spring core, sem risco de supply chain adicional. |
</threat_model>
