<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → Controller | Input externo entra pelo Controller; sem autenticação (projeto didático) |
| Controller → Service | Validações de negócio no Controller não replicadas no Service — ponto pedagógico DIAG-05 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Elevation of Privilege | Validações de negócio apenas no Controller — DIAG-05 intencional | accept | Módulo didático: a falha de segurança DIAG-05 é o ponto pedagógico. Em produção, validações devem estar no service ou no domínio. Comentário DIAG-05 explica a consequência. |
| T-05-05 | Information Disclosure | RuntimeException retorna mensagem de erro completa para o cliente HTTP | accept | Dev-only. Comentário no handler explica que produção usaria ProblemDetail com status codes adequados. |
| T-05-02 | Tampering | @Valid em request bodies — Bean Validation ativo | mitigate | @Valid nos parâmetros de request — Bean Validation valida @NotNull/@NotBlank antes do código do controller executar. |
| T-05-SC | Tampering | Sem novos pacotes neste plano | accept | Todos os pacotes herdados do parent POM já auditados |
</threat_model>
