Estrutura do arquivo:

**Título:** `# Entidades — Aplicadas na Arquitetura Tradicional`

**## O que é** (2-3 parágrafos):
Entidade é um objeto com identidade — dois objetos com os mesmos atributos são objetos diferentes se têm IDs diferentes. Identidade persiste ao longo do ciclo de vida: um aluno que muda de nome é o mesmo aluno. Para teoria completa, ver `docs/02-design-tatico/entidades.md`.

**## Manifestação no módulo camadas (ANTES)**:
Usar `erp-matricula-camadas/src/main/java/br/com/escola/matricula/model/Matricula.java` como exemplo. Snippet de 10-12 linhas mostrando: campos `private UUID id`, `private String status`, e setter `setStatus(String status)` — aceita qualquer string. Sem `equals/hashCode` explícito. Sem método comportamental. Comentário `// ANTES:` no topo. Referência ao arquivo completo.

**## Aplicação no módulo DDD (DEPOIS)**:
Usar `erp-matricula-ddd/src/main/java/br/com/escola/matricula/dominio/modelo/Aluno.java`. Snippet de 12-15 linhas mostrando: `private final AlunoId id` — campo final, identidade imutável. Método `estaAtivo()` com semântica de negócio (não getter booleano `isAtivo()`). Método `desativar()` comportamental (não setter `setAtivo(false)`). Assinatura dos métodos `equals` e `hashCode` baseados APENAS em `AlunoId`. Comentário `// DEPOIS:` no topo.

**## O contraste de identidade** (seção extra curta, 4-6 linhas):
Incluir o contraste pedagógico explícito: "Dois alunos com o mesmo nome são pessoas diferentes (identidade por ID — `equals` usa `AlunoId`). Dois `PeriodoLetivo(2026, 1)` são o mesmo período (Value Object — `equals` usa os valores `ano` e `semestre`). Esta distinção define quando usar Entidade vs. Value Object."

**## O que foi ganho**:
2-3 linhas: `desativar()` encapsula a transição de estado — código externo não pode chamar `setAtivo(false)` e criar um aluno em estado inconsistente. A identidade por ID garante que um aluno transferido de turma (com dados atualizados) é reconhecido como o mesmo aluno.

Regras críticas:
- O exemplo principal de Entidade é `Aluno.java` (tem identidade + ciclo de vida mais simples que Matricula)
- O exemplo de "antes" é `erp-matricula-camadas/model/Matricula.java` (entidade anêmica já documentada em DIAG-02, mas aqui o ângulo é identidade, não apenas ausência de comportamento)
- Manter referência cruzada para `02-entidade-anemica.md` para o estudante que quiser o ângulo DIAG
