**Tabela de distinção Domain Service vs Application Service (padrão PATTERNS.md):**

| Aspecto | Domain Service | Application Service (UseCase) |
|---------|---------------|-------------------------------|
| Onde vive | `dominio/` — sem import de framework | `aplicacao/` — pode usar Spring |
| O que faz | Regra de negócio pura que cruza entidades | Orquestra: busca dados, chama domínio, salva, publica eventos |
| Dependências | Só interfaces de domínio | Usa repositório, event publisher, outros serviços |
| Transação | Não gerencia transação | Pode demarcar `@Transactional` |

**Abertura bottom-up (D-04):** "Como o Aggregate `Matricula` é salvo no banco?" — Se a classe `Matricula` tivesse `@Entity`, `@Table`, `@Column`, ela dependeria do JPA. Mudar de banco de dados exigiria mudar a classe de domínio. A solução: a classe `Matricula` não sabe como é persistida. Uma interface no domínio define o contrato; a infraestrutura cumpre. Dois parágrafos antes de introduzir a interface.

**Snippet da interface no domínio (D-07):** Snippet completo conforme RESEARCH.md TAT-06 — `MatriculaRepositorio` com 4 métodos, zero imports de framework. Comentários:
- `// Java 21: interface pura — zero imports de framework`
- `// DDD fit: domínio define o contrato; infraestrutura cumpre — Regra de Dependência em ação`

**Referência à implementação MyBatis (sem implementar na Fase 2):** Snippet curto mostrando apenas a assinatura `@Repository public class MatriculaRepositorioMyBatis implements MatriculaRepositorio { ... }` com comentário: "A implementação MyBatis completa é entregue na Fase 3 (Implementação). # Resultado esperado: nenhuma ocorrência

**Consequências — O que essa escolha implica:** Subseção curta com positivas e negativas (padrão ADR-001):

Positivas:
- Domínio sem imports de framework — verificável com um grep
- Fácil trocar MyBatis por outra tecnologia de persistência sem tocar no domínio
- Testes de domínio sem banco real — mock da interface

Negativas (trade-offs honestos):
- Mais código de mapeamento escrito manualmente (MatriculaRowMapper)
- Nenhuma auto-geração de queries — cada query em XML ou Java explicitamente
